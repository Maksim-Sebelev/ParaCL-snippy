# ParaCl-snippy
`ParaCl-snippy` - это тестовый генеатор для языка `ParaCL`.

## Зависимости
- ОС: Ubuntu 24.04.3 LTS (Возможна корректная работа и на других ОС, но это не гарантируется)
- Clang 18.1.1 (C++20)
- Ninja 1.11.1+
- CMake 3.28+
- Boost 1.7+

## Установка
```
git clone https://github.com/Maksim-Sebelev/ParaCL-snippy &&
cd ParaCL-snippy &&
cmake -S . -B build -G Ninja -DCMAKE_CXX_COMPILER=clang++ -DCMAKE_BUILD_TYPE=Release &&
cmake --build build
```

## Использование

Исполняемый файл: `./build/paracl-snippy`

Поддерживаемые опции:\
  -h [ --help ]          - *показать информацию о paracl-snippy*\
  -v [ --version ]       - *показать информацию о версии-snippy*\
  -o [ --output ]   arg  - *файл в котором будет сгенерированна программа*\
  -s [ --settings ] arg  - *json файл с настройками генерации*

Для генерации набора тестов используйте bash-скрипт `generate-basic-tests`.\
Как аргумент он принимает директорию в которую будут сохранены сгенерированные тесты

```shell
./build/generate-basic-tests [ouput directory]
```

## Уточнение параметров генератора

Если Вы хотите генерировать какие-то интсрукции чаще или реже других и в целом немного управлять хаосом генерации, то можно задать это через json файл:

```shell
paracl-snippy -s settings.json
```

**Формат json-файла:**

1) Параметр `next-stmt-p` - задается числом от 0 до 1. Задает вероятность генерации следующего стейтмента.\
Например после генерации инсрукции `if` с вероятностью next-stmt-p будет сгенерирована следующая инструкция, и с вероятностью (1 - next-stmt-p) будет завершена генерация. Фактически отвечает за размер программы: чем больше значение, тем в среднем больше сгенерированная программа.
2) Параметр `continue-expr-max-p` - задается числом от 0 до 1. Задает максимальную вероятность расширения экспрешена.\
Вероятность расширение экпрешена - `p` - это параметр, отвечающий за выбор экспрешенов в подобных ситуациях:
Генерируем бинарный оператор +:\
Для левого аргумента:\
С вероятностью `p` будет создан расширяемый экпрешен (бинарный оператор, унарный оператор, оператор вывода на stdout). И с вероятностью `1 - p` будет закончена генерация созданием нерасширяемого экспрешена (Числовой литерал, переменная, оператор ввода с stdin).\
Здесь расширяемый экпрешен - это экспрешен, который может содержать другой экпрешен как аргумент (например бинарный + таковым является, так как левым и правым аргументами могут быть экпрешены. Переменная же таковым не является, так как не имеет аргументов).\
Нерасширяемый экспрешен - это экспрешене, который не содержит другие экпрешены как аргументы (например чисвой литерал таковым является, а унарный минус - нет).

3) Параметр `max-expression-depth` - задается неотрицательным целым числом. Задает максимальную глубину в экпрешенах.\
Например при глубине равной 0, будут генерироваться только терминальные экпрешены (переменные, числа, оператор ввода с stdin (?)). При глубине равно 1, будут сгенерированы только с глубиной рекурсии не более чем 1:

```c
1 + 1; // подходит, глубина рекурсии = 1
1 + (1 + 1); // не подходит, глубина рекурсии = 2 > 1
1; // подходит, глубина рекурсии = 0 < 1
```

3) Параметр `max-scope-depth` - задается неотрицательным целым числом. Задает максимальную вложенность скопов..\
Например при глубине равной 0, будет сгенерирован только глобальный скоп. При глубине равно 1, могут быть сгенерированы while, if, scope с с глубиной рекурсии не более чем 1. Например:

```c
// global scope

while (var < 10)
{
    var += 1;
} // подходит, вложенность = 1

1; // подходит, вложенность = 0 < 1

if (var)
{
    while (var)
    {
        var += 1;
    } // не подходит, вложенность = 2 > 1.
}
```

4) Параметр `statements-weights` - задается массивом из весов стейтментов. Веса задаются целыми неотрицательными числами. Чем больше вес - тем чаще встречается интструкция. Вероятность генерации инструкции = (собственный вес)/(суммарный вес всех инструкций). Если вес = 0, то интсрукция не будет встречаться в коде.\
Имена интсрукций:\
while\
if\
print
expression\
scope\
comment\
semicolon

Дефолтные значение для всех параметров = `1`.

5) Параметр `expressions-weights` - задается массивом из весов экспрешенов. Полный аналог предыдущего параметра `statements-weights` для экспрешенов.\
Имена инструкций:\
binop\
unop\
print\
in\
variable\
number

Дефолтные значение для всех параметров = `1`, за исключением `in`, его дефолтное значение = `0` (чтобы дефолтные тесты не требовали сторонних действий).

7) Параметр `unop-weights` - аналогинен предыдущему `binop-weights`.

8) Параметр `save-div` - задается булевским значением. Нужен для исключения ситуация с делением на 0. Дефолтное значение: `true`

9) Параметр `save-while` - задается булевским значением. Нужен для исключения ситуация с бесконечным while. При правильной работе ParaCL гарантирует не более определенного числа итераций while. Данное количество задается параметром `while-iterations-limit`, а дефолтное значение равно. Использует переменные, даже если они не разрешены остальными параметрами. Дефолтное значение: `true`

10) Параметр `while-iterations-limit` - задается целым неотрицательным числом. Нужен для исключения ситуаций с бесконечным while. Использует переменные, даже если они не разрешены остальными параметрами. **Важно**: при `save-while` = `false` данный параметр ни на что не влияет. Дефолтное значение: `30`

11) Параметр `statements-limit` - задается целыми неотрицательным число. Нужен для строгого размера программы. Задает максимальное количество стейментов в программе. Дефолтное значение: `1000`
